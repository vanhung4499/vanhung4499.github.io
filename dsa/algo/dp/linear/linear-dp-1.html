1. Giới thiệu về Quy hoạch động tuyến tính

Quy hoạch động tuyến tính: Phương pháp quy hoạch động tuyến tính là phương pháp quy hoạch động mà các giai đoạn được chia thành "tuyến tính" (được gọi là "DP tuyến tính").

Nếu trạng thái có nhiều chiều, nhưng mỗi chiều đều được chia thành các giai đoạn tuyến tính, cũng được coi là DP tuyến tính. Ví dụ như bài toán cái túi, DP khoảng cách, DP số chữ số, v.v. đều thuộc loại DP tuyến tính.

Có nhiều cách để chia các bài toán DP tuyến tính.

  • Nếu phân loại theo "số chiều của trạng thái", chúng ta có thể chia các bài toán DP tuyến tính thành: bài toán DP tuyến tính một chiều, bài toán DP tuyến tính hai chiều và bài toán DP tuyến tính nhiều chiều.
  • Nếu phân loại theo "định dạng đầu vào của bài toán", chúng ta có thể chia các bài toán DP tuyến tính thành: bài toán DP tuyến tính đơn chuỗi, bài toán DP tuyến tính hai chuỗi, bài toán DP tuyến tính ma trận và bài toán DP tuyến tính không có chuỗi.

Trong bài viết này, chúng ta sẽ phân loại các bài toán DP tuyến tính theo định dạng đầu vào của chúng và giải thích từng loại bài toán.

2. Vấn đề DP tuyến tính đơn chuỗi

Vấn đề DP tuyến tính đơn chuỗi: Vấn đề có đầu vào là một mảng hoặc một chuỗi duy nhất trong DP tuyến tính. Trạng thái thường được định nghĩa là dp[i]dp[i]dp[i], biểu thị:

  1. Các giải pháp liên quan đến "mảng con kết thúc tại vị trí thứ iii trong mảng (nums[0]…nums[i]nums[0]…nums[i]nums[0]…nums[i])".
  2. Các giải pháp liên quan đến "mảng con kết thúc tại vị trí thứ i−1i - 1i−1 trong mảng (nums[0]…nums[i−1]nums[0]…nums[i - 1]nums[0]…nums[i−1])".
  3. Các giải pháp liên quan đến "mảng con bao gồm iii phần tử đầu tiên trong mảng (nums[0]…nums[i−1]nums[0]…nums[i - 1]nums[0]…nums[i−1])".

Sự khác biệt giữa 3 cách định nghĩa trạng thái này là một phần tử nums[i]nums[i]nums[i].

  1. Trạng thái thứ nhất: Độ dài của mảng con là i+1i + 1i+1, mảng con không thể rỗng.
  2. Trạng thái thứ hai và trạng thái thứ ba: Hai trạng thái này có cùng mô tả. Độ dài của mảng con là iii, mảng con có thể rỗng. Khi i=0i = 0i=0, trạng thái này được sử dụng để biểu thị mảng rỗng (bao gồm 000 phần tử đầu tiên trong mảng).

class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        size = len(nums)
        dp = [1 for _ in range(size)]

        for i in range(size):
            for j in range(i):
                if nums[i] > nums[j]:
                    dp[i] = max(dp[i], dp[j] + 1)
        
        return max(dp)

Ý tưởng 1: Code quy hoạch động
class Solution:
    def maxSubArray(self, nums: List[int]) -> int:
        size = len(nums)
        dp = [0 for _ in range(size)]

        dp[0] = nums[0]
        for i in range(1, size):
            if dp[i - 1] < 0:
                dp[i] = nums[i]
            else:
                dp[i] = dp[i - 1] + nums[i]
        return max(dp)

class Solution:
    def maxSubArray(self, nums: List[int]) -> int:
        size = len(nums)
        subMax = nums[0]
        ansMax = nums[0]

        for i in range(1, size):
            if subMax < 0:
                subMax = nums[i]
            else:
                subMax += nums[i]
            ansMax = max(ansMax, subMax)
        return ansMax

class Solution:
    def longestCommonSubsequence(self, text1: str, text2: str) -> int:
        size1 = len(text1)
        size2 = len(text2)
        dp = [[0 for _ in range(size2 + 1)] for _ in range(size1 + 1)]
        for i in range(1, size1 + 1):
            for j in range(1, size2 + 1):
                if text1[i - 1] == text2[j - 1]:
                    dp[i][j] = dp[i - 1][j - 1] + 1
                else:
                    dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

        return dp[size1][size2]

Ý tưởng 1: Quy hoạch động
1. Phân chia giai đoạn

2. Định nghĩa trạng thái

Định nghĩa trạng thái dp[i][j]dp[i][j]dp[i][j] là: Số lần thao tác tối thiểu để chuyển từ "từ con gồm iii ký tự đầu tiên của word1word1word1" thành "từ con gồm jjj ký tự đầu tiên của word2word2word2".

    1.     2.     3. Xóa (word1word1word1 xóa ký tự thứ i−1i - 1i−1): Số lần thao tác tối thiểu phụ thuộc vào "từ con gồm iii ký tự đầu tiên của word1word1word1" chuyển thành "từ con gồm j−1j - 1j−1 ký tự đầu tiên của word2word2word2", cộng thêm một lần xóa, tức là dp[i][j]=dp[i][j−1]+1dp[i][j] = dp[i][j - 1] + 1dp[i][j]=dp[i][j−1]+1.

4. Điều kiện ban đầu
  • Khi i=0i = 0i=0, "từ con gồm 000 ký tự đầu tiên của word1word1word1" là một từ rỗng, cần thực hiện jjj lần thêm để chuyển thành "từ con gồm jjj ký tự đầu tiên của word2word2word2", tức là dp[0][j]=jdp[0][j] = jdp[0][j]=j.

5. Kết quả cuối cùng

Dựa trên định nghĩa trạng thái, trả về giá trị nhỏ nhất trong tất cả các dp[i][j]dp[i][j]dp[i][j] là kết quả cuối cùng.

Ý tưởng 1: Code
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        size1 = len(word1)
        size2 = len(word2)
        dp = [[0 for _ in range(size2 + 1)] for _ in range(size1 + 1)]

        for i in range(size1 + 1):
            dp[i][0] = i
        for j in range(size2 + 1):
            dp[0][j] = j
        for i in range(1, size1 + 1):
            for j in range(1, size2 + 1):
                if word1[i - 1] == word2[j - 1]:
                    dp[i][j] = dp[i - 1][j - 1]
                else:
                    dp[i][j] = min(dp[i - 1][j - 1], dp[i - 1][j], dp[i][j - 1]) + 1
        return dp[size1][size2]

  • Độ phức tạp không gian: O(n×m)O(n \times m)O(n×m). Sử dụng một mảng hai chiều để lưu trữ trạng thái, vậy tổng độ phức tạp không gian là O(n×m)O(n \times m)O(n×m).
